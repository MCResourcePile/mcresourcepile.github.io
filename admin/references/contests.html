Updates Tags
Each map contest page must have a json data file with the following information as well as a HTML page containing only {{> contest}} in the saved document, nothing else. In addition to the following attributes the page data file must also include a "title": "" and "page-category": {} information like all other pages. Images for the league (including banners and map thumbnails) must be located at assets/img/contest/ with map images in a folder called maps.
Main contest attributes
Attribute Description Value Default
"name": The name of the contest. String
"id": The ID used to reference the contest assets. String
"map-group": The group which the maps referenced in the contest are involved in, connecting the contest to other map pages. Groups include avicus, overcast, pgm, and stratus. Group ID
"heading": Display a text heading in the contest banner. Map contests without custom banners need this to be true. Boolean false
"host": The network or individual who hosted the map contest. String
"anounncement": The page which contains the map contest details. URL
"stages": [] The dates in which each stage of the contest took place. Stage info
"panels": [] Panels which can contain extra information or details about the map contest. This is not required. Optional information
"fisrt_place": The name of the map that finished first. Map name
"second_place": The name of the map that finished second. Map name
"third_place": The name of the map that finished third. Map name
"score_method": The method of scoring used. Can either be percent, which calculates a percentage score out of 100, or sum, which simply adds each criteria score together. Score method sum
"critera": {} The criteria which each map was evaluated on, including any bonus points. Criteria
"submissions": [] The maps which were submitted to the contest. Map info
Stage Information
Provides information about the different stages of the tournament.
Attribute Description Value Default
"stage": The name of the stage being played. String
"date": The starting date for this specific stage. String
Panels
Panels can contain pretty much anything regarding the tournament information that hasn't already been covered. Panels can come in two sizes; full which spans the whole screen width, or half which only covers half the screen width.
Attribute Description Value Default
"label": The label which is displayed as the panel title. String
"size": The size of the panel. Can either be full or half. Option string half
"rows": [] Contains the specifc rows which will be displayed on the panel
Rows sub-attributes
"label": The row label displayed on the left. This is optional and if left blank the content will span the entire row. String
"content": The text displayed inside of the row String
Scoring Criteria
Provides information about the scoring criteria and score bonuses for the map contest. Please note that there can only be one score bonus per contest currently. Support for multiple score bonuses may come in the future.
Attribute Description Value Default
"label": The display label for the score criteria. String
Score attributes (only used for scored criteria)
"max": The maximum score allowed for the score criteria. Number
"weight": The weight for the score criteria. The sum of all the criteria weights must equal 100. Number
Score bonus attributes (only used for scored bonuses)
"operation": The method of score bonus application. Possible operations include add, multiply, and percent. Operation add
"amount": The amount the score bonus is worth. Number 1
"reason": The reason for the score bonus. String
Map Submissions Information
Provides information about the different maps that were submitted to the map contest. Image thumbnails for these maps must be put into a folder called maps inside the contest folder. These images must have the same name as the slug reference ID of the map.
Attribute Description Value Default
"map": The name of the map. String
"slug": The map's unique slug ID, which is generally auto-generated by the associated network and is used to reference external map pages, downloads and XML documents. String
"authors": [] The author(s) of the map. User Data
"gamemode": The map's gamemode. String
"scores": [] The author(s) of the map. User Data
Scores sub-attributes
"criteria": The criteria ID which is being scored. String
"score": The map's score for the specific critera. Can only be used on criteria which is not a score multiplier, and cannot be mixed with "eligible":. String
"eligible": If the map is eligible for a score bonus. Can only be used on criteria which provides a score bonus, and cannot be mixed with "score":. Boolean
Updating the map contest listing
Data for the contest listing pages are seperate from the contests' individual data file and is located within the (network)/index.json file.
Attribute Description Value Default
"name": The name of the map contest. String
"id": The ID used to reference the map contest assets and page. String
"heading": Display a text heading in the contest banner. Map contests without custom banners need this to be true. Boolean false
"disabled": Hide the contest from the public listing. Boolean false
