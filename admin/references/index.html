Main Structure

Contributing guides and references to the ResourcePile website

Every page must have an accompanying data file located within the src/data/... directory in order to be built correctly. Inside this data directory is a mirror structure of the site and it is essential that the data file and HTML file are in the same locations. For example, a HTML file located at src/folder/file.html would have a data file at src/data/folder/file.json.

Please note that the last property of any root, object, or array property must not be followed by a comma. This will cause our JSON validator to throw red flags and will cause the automatic builds to fail.

Main Properties

Name Description Value Default
"title": Main page heading and document title String ResourcePile
"subtitle": A subtitle placed underneath the main heading String
"description": A more detailed description of the page content placed into the meta
Note: If no description is provided, the subtitle will be used instead		 String
"page-category": [] An array containing categories to control such things and navigation activation Array
Example
{
    "title": "Overcast Map Collection",
    "subtitle": "A collection of publicly downloadable Minecraft maps from Overcast Network made for PGM",
    "page-category": ["collections", "maps", "overcast", "pgm_server"]
}

Settings

Settings alter how the page is constructed and what is displayed or not displayed. Through these settings we can add page specific stylesheets or scripts, as well as define variables which change how certain aspects of the page behave when interacted with or shown.

Name Description Value Default
"settings": {} Holds different settings to define page behaviour and displays Object
Setting Sub-properties
Name Description Value Default
"header": Show or hide the heading and subheading module of pages Boolean "true"
"full": Removes page headings and adds splash background to the page
Note: This setting shouldn't be used for pages other than the home page		 Boolean "false"
"alerts": {} Defines the global site alerts Object
"include": {} Defines which stylesheets and script files are being loaded to pages Object
"maps": {} Defines settings for map specific pages Object
"revisions": {} Defines revision modules being loaded onto the page Object
Example
{
    "title": "Overcast Map Collection",
    "subtitle": "A collection of publicly downloadable Minecraft maps from Overcast Network made for PGM",
    "page-category": ["collections", "maps", "overcast", "pgm_server"],
    "settings": {
        "include": { ... }
    }
}

Alerts

Alerts allow globalised messages to be displayed on every page of the site, excluding those part of the Stratus Addon Project, in styled banners. Alerts should only ever be defined in the global.json file. Having alerts in individual data files will cause the global ones to be overwritten.

Alert sub-properties
Name Description Value Default
"enabled": Enables the global alerts Boolean "false"
"messages": [] Defines the specific alerts being displayed Array
Messages Objects
Name Description Value Default
"class": The alert color - can be either info, success, warning, or danger String "info"
"message": The alert message - can contain HTML elements such as links HTML
"dismissible": Allows the alert to be hidden from the users view boolean
Example
{
    "settings": {
        "alerts": {
            "enabled": true,
            "messages": [
                {
                    "class": "info",
                    "message": "This is a global site alert with a <a href="/index">link</a> to the home page!",
                    "dismissible": true
                },
                {
                    "class": "warning",
                    "message": "This is another alert, but this one cannot be hidden...",
                    "dismissible": false
                }
            ]
        }
    }
}

Includes

Includes define what stylesheets and scripts are being added to the head of the specific document.

Include sub-properties
These two properties can be children of another property, "important": {} in order to place those includes before any of the page specific ones. This should only ever be used in the global.json as using it in page data files will cause overrides.
Name Description Value Default
"stylesheets": [] The stylesheets to be included on the page Array
"scripts": [] The scripts to be included on the page Array
Examples
{
    "settings": {
        "include": {
            "stylesheets": [
                "/assets/css/gamemodes.css"
            ],
            "scripts": [
                "/assets/js/gitzip-api.js"
            ]
        }
    }
}
{
    "settings": {
        "include": {
            "important": {
                "stylesheets": [
                    "https://maxcdn.bootstrapcdn.com/font-awesome/4.7.0/css/font-awesome.min.css",
                    "/assets/css/bootstrap.css"
                ],
                "scripts": [
                    "https://ajax.googleapis.com/ajax/libs/jquery/1.11.1/jquery.min.js"
                ]
            }
        }
    }
}
